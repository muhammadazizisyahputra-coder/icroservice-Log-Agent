# 🛡️ Microservice Log Agent 

基于大语言模型与多 Agent 协同的微服务日志异常检测与根因分析助手。

## 🎯 项目痛点
当前微服务架构下，业务链路复杂。一旦出现线上 P0/P1 级报警，开发和运维人员排查跨服务日志链路极其耗时，传统基于规则的正则匹配已无法满足动态排查需求。

## ⚙️ 核心架构与 Agent 流转
本项目采用 LangChain 框架搭建，包含深度的长链推理与 Tool Calling，设计了三个核心 Agent 进行闭环协作：

1. **Collector Agent（收集者）**：监听 Webhook 报警，自动触发并调用外部 API（如 Kibana/Elasticsearch），精准提取发生异常前后 10 分钟的长上下文日志数据。
2. **Diagnoser Agent（诊断者）**：运用大模型长链推理能力，结合历史故障知识库（RAG），分析复杂的错误堆栈并生成根因报告。
3. **Reviewer Agent（审查者）**：基于诊断报告，给出具体的代码修复建议或运维降级指令。

## 📊 运行状态与收益
- **状态**：目前在企业内部网络稳定运行，覆盖 5 个核心微服务。
- **日消耗**：约 600 万 Token（重度依赖长上下文处理能力）。
- **业务收益**：将 P0 级故障的平均定位时间（MTTT）降低了 50% 以上。

> **⚠️ 声明：** > 由于本项目深度耦合公司内部的业务日志脱敏与鉴权网关，核心业务逻辑代码暂不予开源，当前仓库仅作项目架构说明。
