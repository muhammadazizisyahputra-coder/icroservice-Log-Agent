# 中国法律与隐私合规指南（简要）

本文件给出在中国境内处理个人信息（尤其是聊天记录、照片等敏感数据）时的合规建议。**本说明仅为工程实现建议，不构成法律意见。** 在产品上线前，请咨询专业法律意见。

## 适用法律（示例）
- 《中华人民共和国个人信息保护法》（PIPL）
- 《中华人民共和国网络安全法》
- 《数据安全法》（DSL）

## 合规要点（工程实践建议）
1. 明确处理目的与最小必要原则
   - 仅为用户明确同意的目的处理数据（例如“生成一个模拟其说话风格的本地 Skill”）。
   - 避免收集与目的无关的数据。

2. 获取有效同意
   - 在收集聊天记录或导入文件前，应当明确告知用户用途、范围、保存期限、共享范围，并获得明确授权（书面或电子同意）。

3. 本地处理优先、避免外传
   - 优先在用户本地或受控环境中处理个人数据，**尽量不上传到境外或第三方服务**。
   - 若必须使用第三方（如云端模型或 embedding 服务），应当：
     - 告知用户并征得同意；
     - 评估第三方的安全与合规能力；
     - 对敏感数据进行脱敏或去标识化处理（如移除手机号、身份证号等）。

4. 最小化存储与加密
   - 仅保留实现功能所需的数据，并设置合理保留期；
   - 存储时使用适当加密（静态加密，如 AES256），并保护好密钥管理；
   - 传输时使用 TLS/HTTPS

5. 匿名化与去标识化
   - 在可能的情况下对数据进行去标识化（如替换真实姓名、手机号为代号），保留语义信息而移除直接识别信息。

6. 用户权利
   - 支持用户访问、更正、删除其个人信息；
   - 支持用户撤回同意（撤回后应停止处理并删除相关数据）。

7. 第三方合规
   - 与第三方服务商签署合规合同（包括数据处理协议、跨境协议等），确保其处理与存储符合法律要求。

8. 日志与审计
   - 对数据访问与处理记录进行记录与审计，保留必要的追溯链路（同时注意日志中不要泄露敏感信息）。

9. 敏感数据特殊处理
   - 对身份证、银行卡、健康信息等特殊敏感个人信息应当避免收集；若必须收集，须走更严格的合规流程（明示、最小化、加固保护并在法律允许范围内处理）。

10. 合规评估与本地法律咨询
   - 在产品面向公众前，建议进行专项数据保护影响评估（DPIA）。
   - 必要时咨询律师并根据公司/机构合规要求制定隐私政策与用户协议。


## 开发者快速清单（工程实践）
- 将 personalities/ 目录设置为本地存储并在 .gitignore 中忽略（已经设置）。
- 使用环境变量管理密钥（.env + 不提交到仓库）。
- 提供一个 API/命令用于删除指定用户或指定 Skill 的所有本地数据。
- 在 README 和 UI 中明确告知用户数据处理规则与删除方式。
